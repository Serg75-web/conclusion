2. **Перечень необходимых разрешений, данных и доступов.** Перед автотестированием нужно получить письменное разрешение от владельца ресурса на тестирование, а также подписать соглашение о конфиденциальности и согласие на обработку данных.Кроме того, нужно разрешение на доступ к системе администрирования ресурса для проверки поступления данных формы в БД. Разрешение на Negative Testing. 
3. **Перечень и описание возможных рисков при автоматизации.** При автоматическом тестировании формы отправки имеются следующие риски: риск засорения реальных баз данных (в случае использования тестовых заявок), ложных срабатываний (из-за задержек сети и пр.). Также при усложнении формы усложнится и обслуживание составленных тестов — их нужно будет обновлять.
